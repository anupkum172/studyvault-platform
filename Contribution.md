Git Branching Strategy

- main: Stable production-ready code
- develop: Integration branch
- feature/frontend: Frontend development
- feature/backend: Backend development
- feature/authentication: Authentication module
- feature/documentation: Documentation updates

---

Collaboration Workflow

1. Each member works on their assigned feature.
2. Changes are committed regularly with meaningful commit messages.
3. Features are tested before merging.
4. Pull Requests are reviewed by teammates before merging into the main branch.

---

Contribution Goal

All four team members will actively contribute through code commits, documentation updates, testing, and project management to ensure balanced participation throughout the development cycle.
